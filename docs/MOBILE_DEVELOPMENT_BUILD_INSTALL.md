# Mobile Development Build Install

## Scope

Use this guide to create and install a GRWM Expo development build for local emulator QA. The installed app target is `com.grwm.mobile`.

Expo Go is not supported for this QA path. Do not use production Firebase data.

## Package Manager

Use pnpm only:

```bash
pnpm install
```

If pnpm and node are not on PATH in this shell, use the bundled runtime and fallback pnpm binary:

```bash
PATH="/Users/olutayooladeinbo/.cache/codex-runtimes/codex-primary-runtime/dependencies/node/bin:$PATH" /Users/olutayooladeinbo/Documents/IAttend\ 2/.tools/bin/pnpm install
```

## App Identity

- iOS bundle identifier: `com.grwm.mobile`
- Android package: `com.grwm.mobile`
- iOS simulator install check: `pnpm qa:mobile:install-check`
- Android emulator install check: `adb shell pm path com.grwm.mobile`

The identity is defined in `apps/mobile/app.json` and should not be changed for local QA.

## Firebase Emulator Prep

Create local mobile env values from the safe emulator example:

```bash
cp apps/mobile/.env.emulators.example apps/mobile/.env.local
```

For iOS simulator, keep the default host values:

```bash
EXPO_PUBLIC_FIREBASE_AUTH_EMULATOR_HOST=127.0.0.1
EXPO_PUBLIC_FIRESTORE_EMULATOR_HOST=127.0.0.1
```

For Android emulator, use:

```bash
EXPO_PUBLIC_FIREBASE_AUTH_EMULATOR_HOST=10.0.2.2
EXPO_PUBLIC_FIRESTORE_EMULATOR_HOST=10.0.2.2
```

Build Functions before relying on Functions emulator definitions:

```bash
pnpm functions:build
```

The mobile emulator scripts also run `pnpm functions:build` before starting Firebase emulators so `functions/lib/index.js` exists.

Start the standard emulator set:

```bash
pnpm qa:mobile:emulators
```

If another local project owns the standard ports, use the isolated set and align `apps/mobile/.env.local` with Auth `9100` and Firestore `8085`:

```bash
pnpm qa:mobile:emulators:isolated
```

## Route A: EAS iOS Simulator Build

This route requires an Expo account session or `EXPO_TOKEN`. Check auth without exposing tokens:

```bash
test -n "$EXPO_TOKEN"
pnpm --filter mobile exec eas whoami
```

Create an iOS simulator development build:

```bash
pnpm mobile:eas:build:ios-simulator
```

Install and run the latest iOS simulator build from the `development-simulator` profile:

```bash
pnpm mobile:eas:run:ios
```

Confirm the installed app:

```bash
pnpm qa:mobile:install-check
```

## Route B: Local iOS Simulator Build

This route requires local Xcode/iOS simulator tooling. Boot the target simulator first, then run:

```bash
pnpm mobile:dev-client:ios-simulator
```

Confirm the installed app:

```bash
pnpm qa:mobile:install-check
```

## Android Emulator Install

Start an Android emulator and confirm `adb` is available:

```bash
adb devices
```

For a local Android development build and install:

```bash
pnpm mobile:dev-client:android
```

For an EAS Android internal development build:

```bash
pnpm qa:mobile:eas:development:android
pnpm --filter mobile exec eas build:run --platform android --profile development --latest
```

Confirm the installed app:

```bash
adb shell pm path com.grwm.mobile
```

## Start Metro For The Dev Client

After the installed development build exists, start Metro:

```bash
pnpm mobile:start:dev-client
```

The root QA alias is equivalent:

```bash
pnpm qa:mobile:start
```

## Rerun Manual QA

With Firebase emulators running, Metro running, and `com.grwm.mobile` installed, rerun the A-G checklist in `docs/MOBILE_EMULATOR_QA.md`.

Before opening the app on iOS, confirm:

```bash
pnpm qa:mobile:install-check
```

If the check fails with `No such file or directory`, install a development build before rerunning manual QA.

## Current Blockers To Record

- EAS cloud builds are blocked when neither Expo login nor `EXPO_TOKEN` is available.
- Local iOS builds are blocked when Xcode, iOS simulator runtimes, CocoaPods, or signing/native generation tooling fail.
- Android installs are blocked when no emulator is running or `adb` is unavailable.
- Functions emulator definitions require `functions/lib/index.js`, generated by `pnpm functions:build`.
